Topic 1 of 4 — Semantic Layers
The Semantic Layer

Giving machines the ability to understand meaning

Without Semantic Layer:
"Package arrived very late" ≠ "Delayed delivery"
With Semantic Layer (embeddings):
"Package arrived very late" ≈ "Delayed delivery"
Similarity score: 0.94 ← nearly identical meaning!
📐

Embeddings

BERT, Word2Vec. Convert text → 384-dim vectors where meaning = proximity.

🗄️

Vector DB

Pinecone, ChromaDB. Store & search millions of vectors by similarity.

Topic 3 of 4 — Rule vs Dynamic
Rule-Based
Systems

When logic is clear, explicit, and fixed

# Classic rule-based loan approval
if age > 18 and income > 30000:
    approve = True
elif credit_score < 500:
    approve = False
# Transparent. Predictable. But brittle!
✅

Best For

Fixed criteria, regulatory audits, safety systems, small datasets, high explainability needs.

⚠️

Limitations

Cannot adapt to new patterns. Expensive to maintain. Fails on edge cases.

Topic 3 of 4 — Rule vs Dynamic
Dynamic ML
Systems

When patterns are complex and constantly changing

# ML model learns rules from 1M past loans
model = RandomForestClassifier()
model.fit(X_train, y_train)
prob = model.predict_proba(new_customer)
# → 0.87 probability of approval
# Model found 47 patterns humans never wrote!
🚀

Best For

Fraud detection, recommendations, image recognition, NLP — where patterns shift or are too complex.

⚠️

Tradeoffs

Needs labelled training data. Can be a black box. Requires monitoring for bias and drift.

Topic 3 of 4 — Rule vs Dynamic
How to Choose?

A practical decision framework

📋 Choose Rule-Based
✓ Fixed, clear criteria
✓ Regulatory audit needed
✓ Small dataset
✓ Rarely changing patterns
✓ Safety-critical systems
🤖 Choose Dynamic ML
✓ Complex patterns
✓ Large labelled dataset
✓ Patterns evolve rapidly
✓ Needs personalisation
✓ Image, audio, or NLP tasks
💡 Pro Tip
Production systems often combine both. Rules for speed & compliance → ML for accuracy & adaptability.

From Demo to
Production

How real enterprise systems are built

1
Rule Engine (Demo) — Fast keyword matching. Transparent but misses sarcasm and context.
2
Fine-tuned BERT (Production) — Understands context, negation, sarcasm. Runs on GPU at scale.
3
Vector DB Search — Store all embeddings. Query: 'find all complaints similar to this one' — instantly.
4
Combine Both — Rules for speed & compliance, ML for accuracy & intelligence.
Summary

1

3
Rule vs Dynamic is a deliberate design decision based on explainability, data volume, and pattern complexity.

Combine both in production — rules for speed & compliance, ML for accuracy & adaptability at scale.
